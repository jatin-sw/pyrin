1. all works that belong to same bug or feature must be pushed as a single commit. 
   exceptions are described below.

2. changes on files that does not belong to main application package (bshop), or not related to code development, 
   like `locale` and `settings` and `docs` must be committed separately for each type of files and seperated from 
   code changes too. this is needed for the peace of mind of developers to reduce the conflicting chances and another 
   breaking changes.
   
3. all commits must have a well formed english comment that describes what has been changed or added.
   and commits should be in lower case except for object types if needed.
